This file explains the information contained in the input.txt file.  The input.txt file should resemble the seven lines below:

TestSiModel            -Line 1
0 0 1                  -Line 2
1 2	                 -Line 3
10 90                  -Line 4
98 2                   -Line 5
0.1                    -Line 6
40000                  -Line 7
0 0 0                  -Line 8
1 1000                 -Line 9
(The coordinates of the atoms and their bonds should be located starting here if you are annealing an existing model.  If there are no coordinates listed, the program will create a new model.)

Line 1 will become the file name of several files (all with different extensions) generated when the code runs.  In this example, if a new model is created, the file containing the new model𠏋 coordinates will be called TestSiModel.start.  If this file contains a set of atoms that are annealed, the energy data will be written to TestSiModel.e, and the annealed model will be called TestSiModel.anneal.  Any error messages are written to TestSiModel.error.

Line 2 contains, from left to right, a flag for printing bond data (with extension .b), a flag for printing a .vasp file (which can be opened in VESTA as a visual model), and a flag for printing a table of nearest neighbor atoms (with extension .nn).  Set the flag to 1 to create the file for the current model and 0 to not create the file.  If these numbers are anything other than 0 or 1, an error message will be written to the .error file and the program will stop.

Line 3 contains the type numbers for each atom type used (see Tuttle_constants.txt).

Line 4 contains the percentage of bond swaps that will be performed on each type of atom.  If these numbers do not add up to 100, an error message will be written to the .error file and the program will stop.

Line 5 contains the number of atoms for each element used (specified in line 3).  In the example above, this model contains 98 atoms of type 1 and 2 atoms of type 3.  If this line does not correspond to line 3, the program will output an error message to the .error file and stop.

Line 6 will be the kT value for annealing.  If kT is less than 0 or greater than 20, an error message will be written to the .error file and the program will stop.

Line 7 is the number of bond switches you want to do during annealing. If this number is a non-zero integer, the model will anneal.  If this number is not an integer, an error message will be written to the .error file and the program will stop.

Line 8 is the length, width, and height of the periodic box.  By default, these are set to 0, which means that the program will automatically determine the size of the box based on the number of atoms.  If you want to use box dimensions other than the default ones, replace the zeroes with those dimensions.

Line 9 contains, from left to right, the flag for volume relaxations and the number of bond switches before volume relaxation begins.  Set the flag to 1 to relax the volume of the current model and 0 to not relax the volume.  If the model will be annealed, the volume will be relaxed after every accepted bond swap if the flag is set to 1.  If this number is anything other than 0 or 1, an error message will be written to the .error file and the program will stop.  If the number of bond switches is not an integer or if this number is greater than the number of bond switches to be performed, an error message will be written to the .error file and the program will stop.
